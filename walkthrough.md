# 🚀 반도체 후공정(PKG) 사외 교육 추천 시스템 최종 개발 보고서 (Walkthrough)

본 보고서는 **1단계 환경 구성 및 UI 개발(Phase 1)**에 이어, **실제 웹 크롤러 구현 및 실시간 데이터베이스 동기화(Phase 2)**를 완료한 최종 보고서입니다.

---

## 1. ⚙️ 실제 웹 크롤러 구현 내역 (`crawler/scraper.js`)

기존의 모의(Mock) 데이터 파싱 로직을 걷어내고, **Playwright 헤드리스 브라우저**를 이용해 실제 외부 반도체 교육 사이트 3곳의 DOM을 실시간으로 긁어오도록 구현을 완료했습니다.

### 1) IDEC (반도체설계교육센터) 크롤링
- **타겟 URL**: 
  - 석박사 과정: `https://www.idec.or.kr/edu/apply/list/?&type=list`
  - 재직자 과정: `https://www.idec.or.kr/edu/apply/lst2/?&type=lst2`
- **구현 내용**: 
  - `table.tbl2 tr` 요소를 파싱하여 각 과정의 캠퍼스, 강의형태(대면/온라인), 제목, 교육기간, 신청 상태를 수집합니다.
  - 행의 `onclick` 속성에서 고유 상세 번호(`no=XXXX`)를 추출해 신청 바로가기 URL을 동적으로 조합합니다.
  - 수집 개수: **총 40개 강좌**

### 2) SeminarHub (세미나허브) 크롤링
- **타겟 URL**: `https://www.seminarhub.co.kr/search.php?findword=반도체`
- **구현 내용**:
  - 세미나허브 내 "반도체" 검색 결과 페이지의 `li` 목록을 순회하여 유리기판, HBM, 패키징 등 관련 핵심 세미나 정보를 추출합니다.
  - 제목에 있는 날짜 태그(`[3.18]`)와 연도 정보를 정규표현식으로 대조하여 표준 날짜 기간(`2026-03-18 ~ 2026-03-18`)으로 정규화합니다.
  - 수집 개수: **총 6개 세미나·컨퍼런스**

### 3) KPCA (한국PCB반도체패키징산업협회) 크롤링
- **타겟 URL**: `http://www.kpca.or.kr/seminar/cont1`
- **구현 내용**:
  - 교육 안내 게시판의 리스트 카드 내부 텍스트 블록 전체를 분할(split) 및 파싱합니다.
  - `일시: `와 `장소: `로 시작하는 라인에서 각각 일정과 교육장소 정보를 추출합니다.
  - 한글 날짜 형식(예: `2026년 8월 5일(수) ~ 7(금)`)을 표준 YYYY-MM-DD 포맷(`2026-08-05 ~ 2026-08-07`)으로 파싱하는 정규식 변환 엔진을 내장했습니다.
  - 수집 개수: **총 6개 패키징/PCB 실무 교육**

## 2. 🔄 데이터베이스 실시간 자동 동기화 구축

- **자동화 파이프라인**: 
  - `node scraper.js` 실행 시 수집된 실시간 데이터(IDEC 40개, 세미나허브 6개, KPCA 6개 = 총 52개)를 미수집된 타 16개 기관의 Seed 데이터(총 16개)와 병합합니다.
  - 최종 병합본(총 68개 강좌)을 임시 스테이징 파일인 `crawler/staging_courses.json`에 저장함과 동시에, **웹 앱의 기본 데이터베이스 파일인 `web/sample-courses.js`를 자동으로 업데이트**하도록 로직을 설계했습니다.
- **캐시 이슈 해결을 위한 UI 추가**:
  - 사용자가 기존 세팅(Mock 데이터)을 브라우저 `localStorage`에 캐싱해두어 신규 크롤링 데이터가 대시보드에 즉각 반영되지 않는 현상을 방지하기 위해, 관리자 스테이징 탭 하단에 **"데이터베이스 초기화 (Reset)"** 버튼을 추가했습니다. 클릭 시 로컬 캐시가 리셋되고 크롤러가 수집한 68개의 실시간 데이터셋이 즉시 반영됩니다.

## 3. 🌐 로컬 웹 서버 환경 구성 (`web/server.js`)

- **도입 배경**: Chrome 등 모던 브라우저에서는 로컬 파일(`file:///`) 경로로 HTML을 직접 실행할 시, ES Modules(`import/export`)에 대한 보안 CORS 정책으로 인해 스크립트 실행이 차단됩니다.
- **해결 방안**: 
  - 외부 무거운 패키지나 CLI 도구 의존성 없이, Node.js 기본 `http` 및 `fs` 모듈로만 구성된 **경량 로컬 웹 서버(`web/server.js`)**를 구축했습니다.
  - 현재 백그라운드 태스크(포트 5000)로 자동 활성화 완료되었습니다.

## 6. 🎨 투명 로고 아이콘 반영 및 웹 서버 이미지 버그 수정

- **투명 로고 아이콘 추가**: 
  - 좌측 상단 `TSP Education Search` 문구 앞에 사용자가 제공한 마이크로칩 아이콘을 투명 PNG 형식(`chip_icon.png`)으로 가공하여 삽입 완료했습니다.
  - 제공된 마이크로칩 이미지의 배경에 포함되어 있던 가상 투명 그리드(흰색/회색 격자무늬 패턴)를 제거하기 위해, 브라우저 Canvas 2D API와 경계선 플러드 필(Boundary Flood-fill) 알고리즘을 이용해 배경 격자 영역만 선택적으로 투명하게 만드는 스크립트를 작성하여 PNG 포맷으로 정밀 추출 및 저장했습니다.
- **웹 서버 바이너리 파일 서빙 버그 수정 (`server.js`)**:
  - 기존 로컬 개발용 웹 서버에서 파일 응답 시 `res.end(content, 'utf-8')` 방식을 사용하여 PNG/JPG 이미지 등의 바이너리 버퍼 데이터가 문자열 변환 과정에서 유실/불완전하게 손상되는 현상을 수정했습니다.
  - 인코딩 옵션을 제거하여 raw 바이너리 버퍼를 손상 없이 브라우저에 그대로 전송하도록 `res.end(content)`로 변경함으로써, 투명 아이콘이 깨짐 없이 완전하게 표시되도록 조치했습니다.

## 7. 🔍 크롤링 데이터 후공정 연관성 필터링 및 키워드 트렌드 가독성/상호작용 개선

- **후공정 연관성 필터링 (`isBackEndRelated` 도입)**:
  - 수집된 강좌 정보 중 반도체 후공정(PKG/패키징)과 무관한 전공정 설계, 아날로그 레이아웃, 영상 코덱 등의 강좌가 노출되어 대시보드가 오염되던 현상을 수정했습니다.
  - `crawler/scraper.js`에 후공정 핵심 키워드군(패키징, HBM, CoWoS, 본딩, 기판, PCB, 신뢰성, 고장분석, EMC 등)을 정밀 대조하는 `isBackEndRelated` 필터 함수를 탑재하고 크롤링 데이터 병합 시 필터링하도록 설계했습니다. (예: IDEC 수집 40개 강좌 중 3개의 핵심 패키징/신뢰성 강좌만 필터링 수집 성공).
- **Warpage 대표 강좌 영구 시드 이관**:
  - 크롤러 필터링 활성화 시 IDEC 크롤링이 성공하더라도 휨 해석 대표 과정인 `ANSYS/ABAQUS 활용 반도체 패키지 Warpage 해석 실무` 강좌가 항상 노출될 수 있도록 영구 시드(`otherSeeds`) 리스트로 이관 연동을 완료했습니다. (최종 통합 교육 수: 29개).
- **키워드 트렌드 가독성 개편 및 상호작용(필터링) 구현**:
  - 가독성이 낮았던 트렌드 텍스트(#HBM4 등 흰색 텍스트 오버라이드)를 연한 블루 캡슐 배경과 진한 블루 텍스트를 조합한 고대비 세련된 라운드 칩 배지 디자인으로 개편하여 가독성을 비약적으로 높였습니다.
  - 대시보드의 트렌드 칩(예: #HBM, #CoWoS, #Warpage 등)을 클릭하면 자동으로 "교육/행사 탐색" 탭으로 라우팅되고 검색 인풋에 키워드가 채워지며 탐색 그리드가 필터링되는 상호작용 로직(`web/app.js` 내 `initDashboard` 기능 개선)을 탑재했습니다.

## 8. 🎨 Phase 3: UI 개선 및 벤치마킹 기능 (홈 이름 변경, 로고 확대, 카드 필터, Quick Finder 마법사)

사용자의 추가 요구사항과 tsp-people-education 벤치마킹 분석에 따라 4가지 핵심 개선사항을 완벽히 이식했습니다.

### 1) 사이드바 반도체 패키지 로고 크기 확대
- **변경 사항**: `.logo-icon-img` 스타일을 `32px`에서 `48px`로 확대하고, 둥근 모서리 반경을 `8px`로 상향했습니다.
- **효과**: 로고 이미지 내의 격자 패턴과 반도체 패키지(PKG) 형태가 한눈에 명확하게 들어오며, 고유 브랜드 텍스트인 "TSP Education Search"와의 레이아웃 비례감이 매우 완성도 높게 어우러집니다.

### 2) 대시보드 명칭 ➔ "홈" 변경
- **변경 사항**: 
  - 사이드바 네비게이션 내 "대시보드" 라벨을 "홈"으로 변경했습니다.
  - 메인 본문 상단 헤더 타이틀의 초깃값을 "홈"으로 갱신했습니다.
  - 탭 클릭 시 네비게이션 컨트롤러에서 가져오는 탭 텍스트 동기화 처리가 온전히 이루어지도록 로직을 다듬었습니다. (내부 ID인 `dashboard` 탭 라우팅은 보존하여 기존 코드와의 의존성 격리 유지).

### 3) 통계 요약 카드 클릭 시 탐색 탭 연동 및 필터 자동 지정
- **변경 사항**: 
  - 홈 화면의 통계 요약 카드 4종("전체 교육·행사", "국내 과정", "해외 과정", "세미나·컨퍼런스")에 대해 클릭 이벤트 리스너를 결합했습니다.
  - 통계 카드를 누르면 자동으로 "교육/행사 탐색" 탭으로 화면이 이동하며, 해당 조건(국내/해외/세미나)의 필터 단추가 활성화되고 결과 목록 그리드가 실시간 갱신됩니다.
  - 마우스 호버 시의 피드백을 강화하기 위해 `cursor: pointer;` 효과 및 테두리 색상이 Vibrant Royal Blue로 반응하며 부드럽게 반짝이는 호버 스타일을 이식했습니다.

### 4) "나에게 맞는 교육 찾기" (Quick Finder) 4단계 진단 마법사 모달
- **구현 방식**: 히어로 영역의 메인 CTA 단추인 "나에게 맞는 교육 찾기"를 누르면 4단계 대화형 진단 창이 팝업됩니다.
- **진단 과정**:
  - **Q1. 담당 직무** (설계/공정/설비/신뢰성/소재/수율) ➔ **Q2. 경력 연차** (신입/주니어/시니어/전문가) ➔ **Q3. 강화 분야** (설계/공정/소재/신뢰성/트렌드) ➔ **Q4. 희망 일정** (이번달/다음달/분기내/연중)
